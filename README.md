# voting-service-web
Multi-purpose online voting service

## 시스템 개요

`voting-service-web`은 폐쇄형 선거인명부를 기반으로 운영하는 온라인 투표 시스템입니다. 관리자는 선거인명부를 만들고 투표를 생성한 뒤, 투표가 시작되면 명부와 문항을 잠가 투표 진행 중 임의 변경을 제한합니다. 선거인은 투표자 포털에서 선거인 확인을 거친 뒤 투표하거나, 투표 종료 및 결과 공개 후 결과를 열람합니다.

현재 기본 운영 방식은 다음과 같습니다.

1. 관리자가 독립 선거인명부를 생성합니다.
2. 관리자가 투표 제목, 설명, 유형, 일정, 문항, 선택지, 연결 명부를 설정합니다.
3. 투표 시작 전에는 문항과 명부를 수정할 수 있습니다.
4. 관리자가 투표 시작을 누르면 시작 시각이 현재 시각으로 갱신되고 투표가 즉시 진행 상태가 됩니다.
5. 투표가 시작되면 연결된 명부, 문항, 선택지 수정이 제한됩니다.
6. 선거인은 호수번호, 이름, 식별번호, 생년월일로 선거인 명부 확인을 거친 뒤 투표합니다.
7. 투표 제출 후에는 투표 참여가 완료되며 기본 정책상 다시 수정할 수 없습니다.
8. 관리자가 투표를 종료하고 결과 집계, 확정, 공개 절차를 진행합니다.
9. 공개된 결과는 투표자 포털과 관리자 결과 화면에서 확인할 수 있습니다.

## 주요 플로우

### 관리자 플로우

- **선거인명부 관리**: 투표 생성과 분리된 독립 명부를 생성, 수정, 복제합니다. 이미 시작된 투표에서 사용 중인 명부는 수정할 수 없습니다.
- **투표 생성**: 기본 정보, 문항/선택지, 선거인명부 연결을 순서대로 설정합니다.
- **투표 시작**: 별도 추가 인증이나 검수 승인 없이 권한이 있는 관리자가 바로 시작할 수 있습니다. 시작 시점부터 일정은 현재 시각으로 갱신되고 연결 명부가 잠깁니다.
- **투표 운영**: 진행 중 투표는 일시중단, 재개, 종료할 수 있습니다.
- **결과 관리**: 종료 후 집계, 확정, 공개를 진행합니다. 공개 시 입력한 공지 문구는 투표자 결과 화면에도 함께 표시됩니다.

### 투표자 플로우

- **투표 목록 확인**: 진행 중인 투표와 완료된 투표를 투표자 포털에서 확인합니다.
- **선거인 확인**: 호수번호, 이름, 식별번호, 생년월일을 입력해 명부와 대조합니다.
- **투표 제출**: 선택지를 확인하고 제출합니다. 제출 후에는 투표 참여가 완료되며 다시 수정할 수 없습니다.
- **결과 열람**: 관리자가 결과를 공개한 뒤 선거인은 결과를 열람할 수 있습니다.

## 보안 및 개인정보 보호 원칙

이 시스템은 온라인 투표 운영 중 개인정보와 투표 내용을 직접 연결하지 않는 것을 핵심 보안 원칙으로 삼습니다.

### 개인정보 최소 처리

- 선거인 확인에 필요한 최소 정보만 입력받습니다.
- 화면에는 운영에 필요한 정보만 표시하며 내부 식별자, 세션 값, 토큰 원문은 표시하지 않습니다.
- 오류 메시지에는 입력된 개인정보 원문을 다시 노출하지 않습니다.
- 로그와 감사 기록에는 원시 개인정보, 원문 토큰, 원문 세션 값이 남지 않도록 제한합니다.

### 선거인 정보 보호

- 선거인명부의 확인용 식별값은 정규화한 뒤 HMAC 기반 해시로 비교합니다.
- 호수번호는 앞자리 0을 제거해 같은 세대를 같은 값으로 처리하고, 식별번호와 생년월일의 앞자리 0은 보존합니다.
- 명부 원본 데이터는 투표 내용과 직접 연결하지 않습니다.
- 이미 시작된 투표에서 사용 중인 명부는 수정할 수 없어, 투표 중 명부 조작 가능성을 줄입니다.

### 암호화와 해시 처리

- 관리자 비밀번호는 원문 저장 없이 비밀번호 해시만 저장합니다.
- 관리자 세션, 투표자 세션, 초대/인증 관련 토큰은 원문 대신 해시 값을 저장합니다.
- 개인정보 payload는 암호화 저장을 사용해 데이터베이스만으로 원문을 바로 식별하기 어렵게 설계합니다.
- HMAC 키와 암호화 키는 환경변수로 주입하며 코드와 저장소에 포함하지 않습니다.

암호화와 해시는 정보 노출 위험을 낮추는 보호 장치입니다. 운영 환경에서는 키 관리, 접근 통제, 백업 보호, 서버 보안 업데이트가 함께 유지되어야 합니다.

### 익명투표 보호

- `Ballot`, `Vote`, `AnonymousBallotGroup`에는 유권자 ID, 선거인 ID, 투표자 세션 ID 같은 직접 연결 필드를 두지 않습니다.
- 유권자 인증 흐름과 투표 제출 흐름을 분리합니다.
- 관리자 화면과 API 응답에서 특정 유권자의 선택 내용을 알 수 있는 정보를 표시하지 않습니다.
- 같은 PC에서 여러 선거인이 순차적으로 투표해도 각 선거인의 제출이 독립적으로 집계되도록 처리합니다.

### 결과 공개와 무결성

- 투표가 종료되고 관리자가 결과를 집계, 확정, 공개하기 전까지 결과는 투표자에게 공개되지 않습니다.
- 공개된 결과는 직접 덮어쓰지 않고 정정 또는 무효 처리 흐름을 통해 관리합니다.
- 결과 화면은 1표만 있어도 득표수를 표시해 선출 여부를 확인할 수 있도록 합니다.
- 투표하지 않은 선거인은 결과에서 `기권`으로 집계되어 총 결과 수가 선거인명부 기준 인원과 맞도록 처리합니다.

### 권한과 운영 통제

- 관리자 기능은 로그인 세션과 권한 검사를 통과해야 사용할 수 있습니다.
- 투표 시작, 일시중단, 재개, 종료, 결과 집계, 확정, 공개 등 주요 운영 작업은 권한 기반으로 제한합니다.
- 투표가 시작된 뒤에는 문항, 선택지, 명부 수정이 제한됩니다.
- 공개 결과를 조용히 덮어쓰는 경로를 막고, 결과 버전과 변경 이력을 관리합니다.

## 운영 시 주의사항

- 운영 환경의 `SESSION_SECRET`, `ENCRYPTION_KEY`, `HMAC_KEY`, 데이터베이스 비밀번호는 안전한 비밀 관리 수단으로 관리해야 합니다.
- 운영 DB와 스테이징 DB는 분리해야 합니다.
- PostgreSQL은 외부에 직접 노출하지 않고 애플리케이션 네트워크 내부에서만 접근하도록 구성해야 합니다.
- HTTPS reverse proxy 뒤에서 운영하고, 앱 포트는 로컬호스트에만 바인딩하는 구성을 권장합니다.
- 백업 파일에도 개인정보가 포함될 수 있으므로 백업 저장소 접근 권한과 암호화를 별도로 관리해야 합니다.

## Local initial admin bootstrap

Run RBAC seed first, then set one-time bootstrap environment values in your local `.env`.

Required values:

- `BOOTSTRAP_ADMIN_EMAIL`
- `BOOTSTRAP_ADMIN_PASSWORD`
- `BOOTSTRAP_TENANT_NAME`
- `BOOTSTRAP_ORGANIZATION_NAME`

Then run:

```bash
npm run db:seed
npm run admin:bootstrap
```

The bootstrap command stores only the password hash and blocks recreation when an admin already exists. In production, it also requires `BOOTSTRAP_CONFIRM=CREATE_INITIAL_ADMIN`.
