# 온라인 투표 서비스 요구사항 보완 및 추가 제안서

## 1. 제안 개요

초기 요구사항은 관리자 계정, 투표 생성/시작/종료, 결과 확인/출력, 시작 후 수정 금지, 투표 일정/성격/공개 정책/방식/명부, 명부 암호화, 주요 로그 저장을 포함했다. 최종 설계 과정에서 실제 상용 온라인 투표 서비스 운영에 필요한 보안, 감사, 예외 처리, 익명성, 개인정보 보호 요구가 추가로 식별되었다.

본 문서는 초기 요구사항 대비 보완이 필요한 기능과 정책을 우선순위별로 제안한다.

## 5. 법적 효력 모드에서 필요한 기능

법적 효력 투표 모드는 MVP 범위에서 제외한다. 법적 효력 투표는 향후 확장 기능이며, 본인확인, 전자서명, 증빙 보관, 법적 감사 요건이 추가로 필요하다. 현재 MVP의 강한 익명투표 모델과 법적 효력 투표의 증빙 요구사항은 충돌할 수 있으므로 별도 인증/증빙 모드로 설계해야 한다.

| 제안 항목 | 필요한 이유 | 우선순위 | 현재 설계 반영 | 향후 검토 |
| --- | --- | --- | --- | --- |
| 강한 본인확인 | 법적 다툼 대비 | Legal P0 | 제외 | 휴대폰 본인인증, 전자서명 |
| 전자서명/증명서 | 투표 의사 증빙 | Legal P0 | 제외 | 익명성과 충돌 검토 |
| 증빙 보관 정책 | 법정 보존 기간 준수 | Legal P0 | 부분 반영 | 보관 기간 |
| 외부 감사 로그 | 조작 방지 증명 | Legal P1 | 부분 반영 | 타임스탬프/공증 |
| 법적 고지/동의 | 개인정보 및 절차 동의 | Legal P1 | 부분 반영 | 문구 검토 |

## 9. 남은 의사결정 사항

| 항목 | 분류 | 권장 기본값 |
| --- | --- | --- |
| 인증 정책 기본값 | 운영 시작 전 결정 | MVP 기본값은 초대 링크 + 유권자 식별자 확인. 인증코드는 투표별 선택 옵션 |
| 인증코드 재발송 제한 횟수 | 인증코드 옵션 사용 시 결정 | 일정 시간당 제한 |
| 인증 실패 잠금 기준 | 운영 시작 전 결정 | 유권자 식별 실패 제한은 항상 적용. 인증코드 실패 잠금은 코드 옵션 사용 시 적용 |
| 이의제기 접수 기간 | 운영 시작 전 결정 | 결과 확정 전 기본 접수, 공개 후 제한 접수 |

| 감사 로그 다운로드 승인자 | 운영 시작 전 결정 | Auditor 외 1인 승인 |

| 장애 시 자동 연장 여부 | 조직별 설정 가능 | 관리자 승인 기반 연장 |

| 보고서 템플릿 문구 | 베타 운영 중 결정 가능 | 표준 템플릿 우선 |

| DB 긴급 접근 승인자 | 운영 시작 전 결정 | 보안 담당 + 조직 책임자 |
| 로그 보관 기간 | 운영 시작 전 결정 | 감사 로그와 인증/제출 상세 로그의 구체 보관 기간은 운영 시작 전 확정 필요 |
| 백업 데이터 파기 반영 | 운영 시작 전 결정 | 백업 데이터 파기 반영은 별도 운영 정책 필요 |

## 10. 제안서 결론

본 서비스는 단순 설문 도구가 아니라 신뢰 기반 투표 운영 시스템이다. 따라서 초기 요구사항의 기능 구현만으로는 부족하며, 관리자 권한 분리, 익명투표 데이터 분리, 결과 버전 관리, 감사 로그, 개인정보 파기, 장애/정정/무효 절차가 필수 요구사항으로 포함되어야 한다.

다음 단계에서는 본 설계서와 제안서를 기준으로 ERD, API 설계, 화면 정보구조, 구현 계획을 순차적으로 작성하는 것이 적절하다.
